"state" (mathematical definition)

  • any language
  • definition
  • mathematical

A state s is an n-tuple which is an element of the set of all states St.

Construction

Consider the set of all variables Var. Elements of Var include:

Elements of Var
Symbol Description
x The variable x
si The stack pointer of running program i
mi The available memory of operating system i
ti The temperature of CPU i
pi The position of airplane i
wchicago The current weather conditions in Chicago, IL
wfchicago The weather forecast for Chicago, IL
hi,whitney The height of Mount Whitney at time i
... ...

Aside: Some of these variables seem irrelevant to software. However, this is not the case. Every variable is significant to some software unit, and all variables are relevant to every software unit.

Note that Var is an infinite set.

The values function is a total function which maps each variable v ‚àà Var to the set of possible values of v. For example, if x is a boolean variable in a program, then values(x) = { true, false }.

The set of all states St is the cartesian product of the values of all variables in Var:

St = values(x) √ó values(si) √ó values(mi) √ó ...

A state is an n-tuple (where n is the cardinality of Var, which is infinite) which is an element of St.

Expressing states

Since a state is an n-tuple where n is infinite, we cannot explicitly denote a state with a finite number of characters. If we attempted to explicitly denote a state, it would look like:

‚å© x = false, si = 0x0ea8, mi = 2.35 GiB, ... ‚å™

Instead, we use set builder notation to denote subsets of St. For example, the set of all states where x = false is denoted by:

{ s ‚àà St | x = false }

Every subset of St expressed in set builder notation is infinite, since we cannot hope to constrain every variable in Var. The only exception is the empty set ‚àÖ, which results when we use a predicate that is always false. For example:

Significant variables

A variable is significant to a software unit if the software unit could potentially use the variable to improve its quality of service. The relative significance of variables is subjective.

A significance function is a total function from Var √ó Sw to { x ‚àà ‚Ñù | x > 0 }, where Sw is the set of all software units:

significance ‚àà (Var √ó Sw) √ó { x ‚àà ‚Ñù | x > 0 }

For example, the variables pi, wchicago, wfchicago, and hi,whitney are more significant to air traffic control software than they are to video encoding software.

The variable pi is a property of the physical world, a variable which software cannot directly access. However, software can directly access an abstract representation of pi, and we can use sensors to update the abstract representation of pi. If our pi sensor is faulty, the abstract representation available to our software may differ from the actual value of pi.

The software designer decides which variables are significant enough to include in the software. A software designer would not even consider creating air traffic control software without access to pi; the quality of service provided by the air traffic control software would be terrible without pi. However, the variable hi,whitney is arguably less significant, and a designer may choose to ignore hi,whitney for simplistic air traffic control software.

Every variable v ‚àà Var is significant to some software unit sw ‚àà Sw:

‚àÄx ‚àà ‚Ñù, v ‚àà Var (‚àÉsw significance(v, sw) > x)

Relevant variables

A variable is relevant to a software unit in a particular state if it can affect the behavior of the software unit in that state.

More precisely: a variable v ‚àà Var is relevant to a software unit sw ‚àà Sw in a state s ‚àà St if there exists an alternative state s' ‚àà St, where s' differs from s only in the value of v, which would result in a different behavior for sw.

For example, if you are using video encoding software on your home computer in Chicago, IL, then an alternative state s' with a new value of hi,whitney will never alter the behavior of your video encoding software. In this state, hi,whitney is not relevant to the video encoding software.

However, if you are in an airplane flying over Mount Whitney, then an alternative state s' with a higher value of hi,whitney can affect the behavior of your video encoding software if your airplane is about to crash into Mount Whitney and destroy the hardware that the video encoding software is running on. In this state, hi,whitney is relevant to the video encoding software.
